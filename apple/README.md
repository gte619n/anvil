# Anvil ‚Äî Apple clients (macOS first)

Hybrid shell: a SwiftUI app hosting the Anvil web client in a `WKWebView`, same as the Android
app. **The web UI is bundled in the app** (`Sources/Anvil/web`, generated by
`anvild/web/bundle-native.ts`) and served from a local custom scheme (`anvil-app://`), so the
shell + fonts load offline; only the daemon data connection (WS/REST over Tailscale) needs the
network ‚Äî injected as `window.ANVIL_DAEMON_URL`. APNs push + the iOS target layer on later.

## Build & run (no Xcode.app needed)

```sh
# 1. (re)generate the bundled web assets ‚Äî run after any web client change:
cd anvild && bun run build:web && bun run web/bundle-native.ts ../apple/Sources/Anvil/web

# 2. compile / run the macOS app:
cd apple
swift build      # compiles
swift run        # launches the window
```

`swift run` runs the SPM executable directly (uses `Bundle.module` for the bundled web). This is
the canonical local build with the Command Line Tools ‚Äî **no Xcode.app required**.

## Producing a distributable .app

```sh
cd apple
./make-app.sh            # ‚Üí apple/dist/Anvil.app  (bundles web, builds release, signs)
./make-app.sh --skip-web # reuse the existing Sources/Anvil/web bundle (faster)
./make-app.sh --debug    # package the debug build
open dist/Anvil.app
```

`make-app.sh` needs only the Command Line Tools (`swift`, `iconutil`, `codesign`) ‚Äî **no
Xcode.app**. It lays out `Anvil.app/Contents/{MacOS,Resources}`, copies the bundled web client to
`Contents/Resources/web` (served at runtime via `Bundle.main`), compiles the `.appiconset` into
`AppIcon.icns`, writes `Info.plist` (version pulled from `project.yml`), and **ad-hoc** codesigns
the bundle. Ad-hoc signing means no Gatekeeper/notarization ‚Äî fine for local/Tailscale-internal
distribution; recipients may need to right-click ‚Üí Open the first time. Notarized distribution and
iOS (device install, push, TestFlight) require full **Xcode.app** + the Apple Developer Program.

## Configure the daemon URL

Defaults to the Tailscale URL. Override:

```sh
defaults write com.gte619n.anvil anvil.baseURL "https://your-host.ts.net:7701/"
```
