# Anvil — Android client

Hybrid shell: a hardened Kotlin `WebView` (`MainActivity`) hosting the Anvil web client over
Tailscale, plus native modules for push and device setup. **The web UI is bundled into the
APK** (generated by `anvild/web/bundle-native.ts` and served through a `WebViewAssetLoader`),
so the app shell and fonts load offline; only the daemon data connection (WS/REST over
Tailscale) needs the network. Package id `com.gte619n.anvil`, `minSdk 33` / `targetSdk 35`.

> Because the web UI is baked into the APK, **updating `anvild` never updates the phone's UI** —
> the app has to be re-shipped (a full release rebuilds and redistributes it). See
> [`docs/CI-CD.md`](../docs/CI-CD.md).

## What's inside

| File | What it does |
|------|--------------|
| `MainActivity.kt` | The hardened WebView host — asset-loader for the bundled web client, `window.AnvilNative` bridge, file-chooser + camera/mic permission plumbing, edge-to-edge insets, device-Back wired to the web back-stack. |
| `AnvilMessagingService.kt` | FCM service — registers the device token with the daemon (`POST /api/push/fcm/register`) and shows notifications. Pushes are data-only so they fire while backgrounded; a `kind:"clear"` push dismisses a session's reminder when it's answered elsewhere. |
| `Notifications.kt` | Builds the session-keyed notifications (deep-link on tap; Allow/Deny action buttons on permission pushes). |
| `PermissionActionReceiver.kt` | Handles the Allow/Deny notification actions → `POST /api/permission/respond` without opening the app. |
| `AdbWifi.kt` | Discovers this device's wireless-debugging endpoints over mDNS (`_adb-tls-connect` / `_adb-tls-pairing`) for the in-app ADB-over-Tailscale flow. |
| `Net.kt` | Tiny JSON POST helper used by the push/permission bridges. |

## Configure the daemon URL

The shell loads `BuildConfig.ANVIL_BASE_URL`, defaulting to the hub's MagicDNS HTTPS URL.
Override per machine in `local.properties` (not committed):

```properties
ANVIL_BASE_URL=https://your-host.ts.net:7701/
```

## Build & run (dev)

Requires **JDK 21** + the Android SDK (and **Bun** on `PATH`, for the web bundle). From the
repo root:

```sh
./gradlew :app:assembleDebug        # → app/build/outputs/apk/debug/
./gradlew :app:installDebug         # install onto a connected device/emulator
```

The build's `bundleWebAssets` task automatically runs `bun run build:web` + `bundle-native.ts`
into `app/src/main/assets/web`, so a fresh checkout picks up the current web client with no
extra step. To regenerate the assets by hand (e.g. iterating on the web UI without a full
Gradle build):

```sh
( cd anvild && bun run build:web && bun run web/bundle-native.ts ../app/src/main/assets/web )
```

Builds sign with the **committed debug keystore** (`anvil-debug.keystore`, fixed password
`android`) so a locally-built APK and a CI-built one share a certificate — no
uninstall/reinstall when swapping between them. FCM needs a `google-services.json`
(`com.google.gms.google-services` plugin).

## Shipping

Android ships as a **signed debug APK to Firebase App Distribution** on every merge to `main`
(the `android` job in [`.github/workflows/release.yml`](../.github/workflows/release.yml)) — not
to the Play Store. The Play Publisher plugin (`com.github.triplet.play`) and a real upload
keystore are configured but **dormant**: no workflow invokes them, and they only come into play
if a public-store launch is ever wired. Details: [`docs/CI-CD.md`](../docs/CI-CD.md).

## Versioning

`versionName` is `MAJOR.MINOR.<build>` — `MAJOR.MINOR` from the repo-root
[`VERSION`](../VERSION) file (shared with iOS, the macOS client, and the server), the build
number from CI's `ANVIL_BUILD_NUMBER`. Local builds report `MAJOR.MINOR.0`. Bump `VERSION` to
start a new line.
